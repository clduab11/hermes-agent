- **Async/Await**: Non-blocking I/O throughout
- **Batch Processing**: Multiple documents in parallel
- **Model Caching**: LEGAL-BERT loaded once
- **Graph Caching**: Citation graph cached in Redis
- **Query Caching**: Database results cached

### Bottlenecks & Solutions

| Bottleneck | Impact | Solution |
|------------|--------|----------|
| LEGAL-BERT Inference | High CPU | GPU acceleration, model quantization |
| Database Queries | I/O bound | Connection pooling, caching |
| Citation Graph | Memory | Incremental loading, lazy evaluation |
| API Rate Limits | External | Retry with backoff, multi-provider |
